# Руководство по запуску бенчмарка

В данном репозитории реализован кастомный адаптер для запуска бенчмарка (в частности IFEval) в обход стандартного OpenAI SDK, работающий через прямые HTTP запросы к вашему API.

## Конфигурация окружения (.env)
Перед началом работы убедитесь, что в корневой папке проекта создан файл `.env`, в котором прописаны все необходимые настройки:
```env
DEFAULT_BASE_URL=базовый_адрес_вашего_API
DEFAULT_TOKEN=ваш_токен_здесь
DEFAULT_MODEL_ID=id_вашей_модели
GEMINI_MODEL_ID=google:gemini-3.1-pro-preview
```

## Описание основных скриптов

### 1. Запуск тестирования
* **`eval.py`** / **`model_eval.py`** — это варианты основного скрипта для отправки запросов к модели, получения ответов и их оценки бенчмарком. 
Скрипт делает асинхронные вызовы и сохраняет результаты по мере их поступления, в том числе генерируя итоговый `.csv` файл.

**Как запустить тестирование:**
```bash
python eval.py
```
* `--limit` — количество тестов за один запуск.
* `--start_index` — индекс, с которого начинается тестирование (смещение), что позволяет разделять тесты на батчи.
* `--temperature` — опционально, температура генерации (по умолчанию 0.0).
* `--concurency` — количество асинхронных запросов генерации.

### 2. Подсчет метрик
* **`eval_metrics.py`** — скрипт для расчета итоговых метрик по полученным результатам (считает Prompt-level и Inst-level Strict/Loose Accuracy).

**Как запустить подсчет метрик:**
(В самом скрипте укажите путь к нужному `.csv` файлу или отредактируйте его запуск из консоли)
```bash
python eval_metrics.py
```

### 3. Извлечение ошибок
* **`compare_process/elen_errors.py`** — утилита, которая парсит итоговый `.csv` файл, находит строки, где модель не справилась (`prompt_level_strict_acc == False`), и сохраняет их в отдельный файл для дальнейшего анализа.

**Как запустить:**
```bash
python compare_process/elen_errors.py
```

## Где сохраняются результаты
Сырые логи бенчмарка в формате `.jsonl`, а также агрегированные результаты в красивом `.csv` формате сохраняются в папку `results_plain/` (или указанную вами через `--output_path`).
